# Siraja Backend

Siraja is a multi-tenant Quran education and memorization platform. This Replit environment runs the **NestJS backend API** only — the Flutter mobile frontend is not runnable here.

## Stack

- **Runtime:** Node.js 20 + NestJS (TypeScript)
- **Database:** MongoDB Atlas (Mongoose)
- **Cache / Queues:** Redis + BullMQ
- **Auth:** JWT (access + refresh tokens), Google OAuth, Apple Sign-In
- **AI:** Moonshot AI
- **Storage:** S3-compatible (Cloudflare R2)
- **Email:** SMTP (nodemailer)

## How to run

The **"Start application"** workflow starts the API:

```
cd backend && nest start
```

The server listens on **port 5000** and is available at `/api/v1`.

### Health check

```
GET /api/v1/health
```

Returns `{ status: "ok", dependencies: { mongodb: "connected" } }`.

## Environment variables

Non-secret values are set in `.replit` (`[userenv.shared]`). Secrets are stored as Replit Secrets:

| Secret | Purpose |
|---|---|
| `MONGODB_URI` | MongoDB Atlas connection string |
| `JWT_ACCESS_SECRET` | Signs access tokens (15 min expiry) |
| `JWT_REFRESH_SECRET` | Signs refresh tokens (30 day expiry) |
| `REDIS_URL` | Redis for BullMQ queues and caching |
| `EMAIL_PASS` | SMTP password for noreply@siraja.website |
| `MOONSHOT_API_KEY` | Moonshot AI for learning intelligence |
| `STORAGE_ACCESS_KEY_ID` | S3/R2 access key |
| `STORAGE_SECRET_ACCESS_KEY` | S3/R2 secret key |
| `STORAGE_ENDPOINT` | S3/R2 endpoint URL |
| `STORAGE_BUCKET` | S3/R2 bucket name |

Google OAuth (`GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`) and Apple Sign-In credentials are optional — those auth flows won't work without them.

## Database seeders

Run these once against a fresh database to populate initial data:

```bash
cd backend
npm run seed:permissions   # Permission/role definitions
npm run seed:quran         # Quran text data
npm run seed:beta-demo     # Demo tenant + sample users
```

## Project structure

```
backend/
  src/
    modules/       # Feature modules (auth, quran, students, memorization, …)
    shared/        # Email, storage, guards, decorators
    database/      # Seeders, Mongoose config
    config/        # configuration.ts (env var mapping)
  dist/            # Compiled output (generated by nest build)
```

## User preferences

- Keep the existing NestJS modular architecture — do not restructure.
- All email calls in auth use-cases must be wrapped in try/catch (non-fatal pattern).
- JWT user id claim is `sub`, not `id` — guards/resolvers must read `request.user.sub`.
